## 1. Создание базы данных  

Создать базу данных **TODO list**, состоящую из следующих таблиц:  
- **Проект**  
- **Задача**  

### Поля сущностей  

- **Проект**  
  - Название проекта 

- **Задача**  
  - Название задачи  
  - Статус задачи (Открыта, Завершена, Выполняется, Отменена)
  - Важность задачи (От 1 до 5)

### Требования к структуре БД  
- У проекта может быть **много задач**, у задачи **только один проект**.  
- Использовать **латинские** буквы для названий таблиц и столбцов.  
- Использовать **внешние ключи** и **ограничения** там, где это необходимо.  
- Для генерации **первичных ключей** использовать **последовательности**.  
- **Первичный ключ должен быть суррогатным**.

## 2. Spring Boot приложение  

Реализовать **Spring Boot** приложение с **REST API**, обладающее следующим функционалом:  

### 1. Создание проекта  
**`POST /api/projects`**  
- Должен вернуть **201 Created** в случае успешного создания.  
- В теле ответа должна быть **сущность проекта с сгенерированным ID**.  
- **Тело запроса**: JSON с данными проекта.
- В ответе должен быть представлен заголовок `Location`, содержащий адрес созданного ресурса.

### 2. Модификация проекта  
**`PUT /api/projects/{projectId}`**  
- Должен вернуть **200 OK** в случае успешного обновления.  
- Если проект **не найден**, вернуть **404 Not Found**.  
- **Тело запроса**: JSON с новыми данными проекта.  

### 3. Удаление проекта  
**`DELETE /api/projects/{projectId}`**  
- Должен вернуть **200 No Content**.
В теле ответа отправить текст: Сущность удалена.

### 4. Получение проекта  
**`GET /api/projects/{projectId}`**  
- Если проект **не найден**, вернуть **404 Not Found**.  

### 6. Получение списка всех проектов  
**`GET /api/projects`**

## 3. Технические требования  
- Приложение должно быть разделено на уровни:  
  - **Controller**  
  - **Service**  
  - **Repository**  
- Для взаимодействия с БД использовать **`JDBCTemplate`** или **`NamedParameterJdbcTemplate`**  
- Приложение должно **корректно обрабатывать** ситуации, когда сущности **не найдены** (например, возвращать `404 Not Found`).
